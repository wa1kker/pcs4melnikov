Архитектура приложения и используемые компоненты
Структурные элементы интерфейса
MaterialApp - корневой компонент, реализующий принципы материального дизайна

Scaffold - базовый каркас страницы с областью заголовка и основным содержимым

AppBar - верхняя панель навигации с отображением названия приложения

Center - элемент для центрирования контента по обеим осям

Column - организация дочерних компонентов в вертикальной последовательности

Padding - создание внутренних отступов для визуального разделения

Интерактивные и информационные элементы
Text - отображение числовых значений счётчика и текстовых подсказок

ElevatedButton - кнопки с визуальным эффектом приподнятости для управления счётчиком

Container - многофункциональный контейнер для стилизации и компоновки

SizedBox - фиксированные промежутки для контроля расстояний между элементами

Элементы визуального оформления
BoxDecoration - комплексное оформление контейнеров (цветовое решение, скругления, границы)

Border - определение стиля обрамления элементов

BorderRadius - настройка радиуса скругления углов

Механизм управления состоянием
Реализация через StatefulWidget
Приложение построено на основе StatefulWidget с выделением отдельного класса состояния _CounterScreenState, что обеспечивает эффективное управление изменяемыми данными.

Принцип работы setState()
Обновление пользовательского интерфейса происходит через вызов метода setState(), который инициирует перерисовку виджета при изменении данных.

Бизнес-логика приложения
Инициализация: начальное значение счётчика устанавливается в 0

Изменение состояния: реализовано три метода управления:

incrementCounter() - инкремент на единицу

incrementByTen() - увеличение значения на 10

resetCounter() - возврат к нулевому значению

Пользовательские взаимодействия
Обработка действий
Короткое нажатие на кнопку "Увеличить" - увеличивает значение на 1

Долгое нажатие на кнопку "Увеличить" - добавляет 10 к текущему значению

Нажатие на кнопку "Сбросить" - обнуляет счётчик

Визуальный отклик
Динамическое обновление числового значения после каждого действия

Цветовая дифференциация функциональности (зелёный для увеличения, красный для сброса)

Визуальная индикация состояния кнопок при взаимодействии

Особенности реализации
Пользовательский опыт
Интуитивно понятная навигация и управление

Наличие скрытой функциональности (долгое нажатие)

Мгновенная визуальная реакция на действия

Чёткое разделение между логикой приложения и представлением
